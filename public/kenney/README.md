# Kenney (CC0) art drop-ins

Drop free **CC0** art from [kenney.nl](https://kenney.nl/assets) into this folder
under the exact filenames below, **add the filename to `manifest.json`**, and the
game uses it automatically. Any missing/unlisted file is safe: that slot falls back
to its procedural shape (enemies/projectiles/pickups) or the placeholder paver
(floor). This mirrors the existing optional-art pattern (`src/ui/playerSheet.ts`,
`public/textures/`).

**`manifest.json`** lists which files are actually present (ships as `[]`). The
loader requests *only* listed files, so an unfilled slot is never fetched — this
keeps optional art truly optional and the production smoke test clean. Example
after dropping in a floor tile and the boss sprite:

```json
["floor.png", "boss.png"]
```

Pipeline: `src/ui/kenneyAssets.ts` (`loadKenneyAssets` in `BootScene.preload`,
`applyKenneyAssets` in `BootScene.create`). The manifest there is the source of
truth — these filenames must match it.

> **Heads-up:** this dev sandbox's network policy blocks `kenney.nl`
> (`403 host_not_allowed`), so the files can't be fetched from inside a Claude
> Code web session. Download them on a networked machine, drop them in here, and
> commit. **Record every file's source URL + license in `CREDITS.md`** (standing
> rule: CC0 / public-domain only unless a license is explicitly cleared).

## How the slots work

- **Enemies** are directional. One front-facing source is baked onto the three
  facing keys (`${base}_down|_up|_side`); left is derived from `side` via flipX
  (the same compromise the roster girls use). For a true multi-frame walk cycle,
  use the sheet pattern in `src/ui/playerSheet.ts` instead.
- **Projectiles/pickups** are single, non-directional textures loaded straight
  onto their key. White-drawn projectiles (pocky/shuriken/bullet) are tinted by
  the weapon colour in-engine, so supply them white/grayscale on transparent.
- **Floor** replaces the placeholder paver under the same texture key, then rides
  the existing perspective recession. Remember to flip `RENDER.floorTexture`.

## Player characters

The 3 playable characters use a separate pipeline (`src/ui/playerSheet.ts`,
`public/roster_sheet.webp` + `public/goth_girl.webp`) because they need real
multi-frame walk cycles. Per the project bible, their final art is generated by
AI image tools at a ¾ Stardew-tilted angle in the neon Edgerunners palette — not
sourced from Kenney.
